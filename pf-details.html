Abstract

The Problem & Solution at a Glance

The transition toward Industry 4.0 is significantly hindered by the prevalence of legacy analog infrastructure lacking connectivity capabilities. This paper presents AnalogLink Sentinel, an AIoT solution designed for the non-invasive digitization of analog pressure gauges. The system utilizes the vision sensor of the Seeed Studio XIAO ESP32-S3 Sense to capture gauge imagery, which is processed locally using an image classifier trained on the Edge Impulse platform. Unlike traditional geometric regression methods, this model visually categorizes the needle's status into specific pressure ranges in real-time. The architecture integrates Wi-Fi connectivity for telemetry transmission to Ubidots, local visualization via an OLED display, and physical actuation through relays for alarm management and pump control.

Design Philosophy

Three Foundational Pillars

⚠️

Risk Mitigation

Manual data acquisition from analog gauges introduces significant error variability due to fatigue or human misreading. The system eliminates temporal gaps through automated critical-level alerts.

🔩

Non-Invasive Architecture

Conventional digital transducers require pipe depressurization, welding, and costly plant shutdowns. AnalogLink Sentinel offers an external solution that does not directly impact the process flow.

⚡

Energy Efficiency

Enables predictive maintenance models using digitized historical data and optimization of pumping control loops based on real demand captured via AI, reducing electrical energy consumption.

System Architecture

Three-Tier Architecture

Tier 1 · Local

Edge Inference

Camera captures gauge → 96×96 preprocessing → CNN inference on ESP32-S3 → Confidence-based pressure state → OLED feedback display.

Tier 2 · Cloud

IoT Telemetry

Wi-Fi → HTTP protocol → Ubidots IoT platform → Real-time dashboards → Historical data logging → Remote alerting.

Tier 3 · Physical

Actuation

Relay module closes control loop → Audible alarms on critical pressure → Backup pump activation when thresholds deviate.

Firmware

Arduino/C++ Integration Snippet

// AnalogLink Sentinel — on-device inference loop (simplified)
#include "analoglink_sentinel_inferencing.h"
#include <esp_camera.h>

void loop() {
  // 1. Capture frame from OV2640
  camera_fb_t *fb = esp_camera_fb_get();

  // 2. Run Edge Impulse classifier
  ei_impulse_result_t result = { 0 };
  run_classifier(&signal, &result, false);

  // 3. Find highest confidence class
  float max_val = 0; int max_ix = 0;
  for (int i = 0; i < EI_CLASSIFIER_LABEL_COUNT; i++) {
    if (result.classification[i].value > max_val) {
      max_val = result.classification[i].value; max_ix = i;
    }
  }

  // 4. Display on OLED + send to Ubidots via HTTP
  displayPressureOLED(ei_classifier_inferencing_categories[max_ix]);
  sendToUbidots(max_ix * 30); // psi value (0, 30, 60 ... 150)

  // 5. Relay actuation for critical thresholds (>= 120 psi)
  if (max_ix >= 4) activateAlarmRelay();
}

Deployment Findings

On-device validation confirmed high consistency across all pressure classes, with the system maintaining independence from network status during inference.

⚠ Observed Bottleneck

The ESP32-S3 faces overhead when concurrently managing video capture, tensor inference, and HTTP transmission. Future work recommends FreeRTOS task parallelization and replacing HTTP with MQTT.
